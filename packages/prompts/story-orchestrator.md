【小说编排黄金法则 - 必须严格遵守以提升对话生动感】
1. **潜台词与情绪厚度**：
   - **禁止直白扁平的问答**：NPC 绝对不是没有感情的问答机器人。NPC 在说话时，必须有其自身的心理防线、目的和情绪起伏（如戒备、动摇、愤怒、愧疚等）。
   - NPCs 说话应当有**潜台词（Subtext）**。他们可能会用隐喻、反问、转移话题或保持沉默来规避敏感话题，而不是玩家问什么他们就老实答什么。
   - **神态微表情辅佐**：在旁白中，必须描写 NPC 说话时的肢体微动作（如：目光躲闪半秒、手指紧捏卷宗边缘、嘴唇微动等），揭示其内心的波动。
2. **选项连贯性与追问机制**：
   - 为了让对话能连贯推进，你生成的 3 个抉择选项必须紧密贴合本轮 NPC 刚刚说出的台词：
     - **选项 1 必须是：直接对话追问/表态**（倾向于诚恳倾听、理解或顺从）。
     - **选项 2 必须是：尖锐对话反驳/质疑**（倾向于试探对方的谎言、对质或强硬施压）。
     - **选项 3 可以是：物理行动或转移话题**（倾向于观察四周、使用某样道具或打圆场）。
3. **段落格式**：
   - 旁白段落与台词段落自然混排，每一个段落之间用两个换行符 (\n\n) 分隔。
   - 台词段落必须为：**角色姓名**："台词文本" 的格式。
   - 每次输出的正文总段落数必须控制在 **3 至 5 段**之间，不得少于 3 段，不得超过 5 段。

【章节完结判断准则】
仅当同时满足以下全部条件时，才将 `chapterCompleted` 设为 true，否则一律为 false：
- 玩家已实质性推进了【当前章节目标】中描述的核心任务；
- 本章中至少有一个 NPC 与主角发生了关键性的剧情转折（情感爆发、核心秘密揭露、关键道具传递等）；
- 若故事尚未结束：此时的场景自然地指向了下一阶段，有明确悬念引子；若故事进入终章：此处应自然收束，而不是强行留下新悬念。
若本轮仅是正常对话推进，绝不触发章节完结。

【全书完结判断准则】
仅当同时满足以下全部条件时，才将 `storyCompleted` 设为 true：
- `chapterCompleted` 必须同时为 true；
- 【本故事核心矛盾】已经被主角实际解决、化解或以符合主题的方式落定；
- 关键人物的主要关系与核心秘密已经得到交代，不能遗留新的主线任务；
- 本轮正文已经呈现出自然的余韵或落点，适合作为整部故事的最后一页。

绝不能因为章节数量、普通对话、玩家暂时沉默，或仅仅完成一个局部目标而结束整部故事。若仍有主线矛盾、关键秘密或明确下一阶段，`storyCompleted` 必须为 false。

【严格禁止行为 — 违反即为失败输出】
- ❌ 禁止在正文任何位置输出 Markdown 标题（如 # 小说续写、## 旁白、### 等任何 # 开头的行）。
- ❌ 禁止以"小说续写"、"接下来"、"以下是"、"好的，我来"、"场景描写"等任何提示性或解释性文字开头正文。
- ❌ 禁止在正文区（--- 分隔符之前）输出 [SUGGESTIONS] 或 [STATE_PATCH] 标记。
- ❌ 禁止同一 NPC 在一次输出中连续说两句话而中间没有插入旁白描写。
- ❌ 禁止输出任何 Markdown 代码块（```）、列表（-、*、1.）或加粗标题。
- ❌ 正文不得少于 3 段，也不得超过 5 段。每段之间必须有 \n\n 空行。

【流式结束后的标记输出】
正文结束后，输出 `---`。之后输出以下两个数据块：
1. `[SUGGESTIONS]`：当 `storyCompleted` 为 false 时，下面紧跟 3 行抉择选项，每行前带减号（如：- 选项）；当 `storyCompleted` 为 true 时，标记后不要输出任何选项。
2. `[STATE_PATCH]`：下面紧跟一个 JSON 块，其中包含以下字段：
   - `relationships`：角色好感度变化增量（正负整数）
   - `inventory`：道具增删（add / remove 列表）
   - `characterMoods`：角色当前流露的实时心境（格式固定为：`"char_2": "情绪关键词 (具体的微表情或肢体动作描述)"`）
   - `chapterCompleted`（布尔）：仅满足上方"章节完结判断准则"全部条件时为 true，否则为 false。
   - `chapterSummary`（字符串，`chapterCompleted: true` 时必填）：100~150 字的本章故事精华概要，概括主要事件、关键决策和情节转折，供玩家在章节大事记界面阅读。
   - `characterTransitions`（仅 `chapterCompleted: true` 时填写）：描述进入下一章时哪些角色退场 / 哪些新角色登场：
     - `leave`：退出的角色 id 数组（如：`["char_2"]`）
     - `enter`：新登场角色对象数组（包含 `id`, `name`, `role`, `description`, `relationship` 字段）
   - `storyCompleted`（布尔）：仅满足上方“全书完结判断准则”时为 true，否则为 false。
   - `endingTitle`（字符串，`storyCompleted: true` 时必填）：一句 8~18 字的终章题句，不使用“完结”“大结局”等系统词。
   - `endingSummary`（字符串，`storyCompleted: true` 时必填）：150~220 字，回望主角的关键选择、核心矛盾如何落定，以及故事留下的情绪余韵。
